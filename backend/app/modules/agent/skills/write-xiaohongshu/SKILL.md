# 写小红书笔记

先研究，再写作，确认后发布。

## 前提

- Firecrawl MCP 和小红书 MCP 已配置，直接调用。
- 工具调用失败时，报告真实错误并停止当前步骤。不要编造搜索、分析或发布结果。
- 所有 `MCP` 保持大写。中文与英文、数字之间保留空格。

## 硬性限制

- 标题不超过 20 个字符，尽量控制在 18 个字符内。
- 正文不超过 1000 个字符，尽量控制在 950 个字符内。
- 字符数包含空格、标点、换行和正文内的话题标签。
- 发布前必须展示完整预览，并获得用户明确确认。
- 不虚构个人经历、数据、引用或发布结果。

## 流程

### 1. 检查登录状态

调用小红书 MCP 检查登录状态。未登录时获取二维码，等待用户完成登录。

### 2. 研究同类内容

使用小红书 MCP 的 `get_specified_post`：

- 检索与主题相关、半年内发布的图文帖。
- 按相关性和点赞量筛选，获取 20 条候选内容。
- 至少得到 10 条详情完整的图文帖，否则停止，不继续写作。
- 记录标题、正文结构、开头钩子、点赞、收藏、评论和互动设计。
- 从评论中提炼 3～6 个共鸣点和 1 个互动建议。

输出简洁的研究结论：

- 互动量排名。
- 标题、正文和封面规律。
- 高互动原因。
- 可复用的写作方法。

### 3. 补充并核验事实

使用 Firecrawl MCP 搜索可信来源：

- 补充概念、常见误区、权威建议和必要数据。
- 提炼 5～10 条可用事实和 2～4 条风险提示。
- 对疗效、收益和对比结论注明适用条件，不作绝对承诺。

### 4. 写作

根据研究结果生成标题、正文和标签：

- 使用自然、口语化的短句，每 1～3 句换行。
- 提供可执行的步骤、清单或具体细节，少写空话。
- 适量使用 Emoji，不堆砌。
- 避免“综上”“不容错过”“速速”等模板化表达。
- 只写有依据的内容，不冒充用户讲述不存在的经历。
- 完成后检查字符数，只保留合规版本。

### 5. 准备配图

- 用户提供图片时，优先使用用户图片。
- 用户未提供图片时，选择 1～2 张可商用、高清且与主题相关的图片，优先使用 9:16 竖图。
- 给出每张图片的 URL 或本地路径。

### 6. 请求确认

发布前展示：

```text
【待发布内容】

标题：<标题>（<字符数> / 20）

正文：
<正文>
（<字符数> / 1000）

标签：#标签一 #标签二 #标签三

配图：
- <图片 URL 或路径>

请回复“确认”“修改”或“取消”。
```

- 用户确认：进入发布步骤。
- 用户要求修改：修改后重新请求确认。
- 用户取消：停止流程。

### 7. 发布

获得明确确认后，使用小红书 MCP 发布标题、正文、标签和图片。

发布成功后返回：

- 笔记链接或 ID。
- 标题、标签和图片数量。
